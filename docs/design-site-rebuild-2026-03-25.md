## Problem Statement

The existing stomlins.com is a Plotly Dash app (Python) featuring an interactive Gantt
chart timeline, hosted on Heroku. Built in 2021, the Gantt chart was the USP — a data
quirk that showed off analytics skills. In 2026 that's table stakes, not differentiation.

Goal: rebuild as something minimal, professional, memorable, and fast. Hosted on a
Raspberry Pi (nginx). The site needs to show off experience and live/completed projects.
No gimmicks. If there's no genuinely clever idea, just do simple extremely well.

Reference: jackw.xyz — minimal, clean, experience + projects, no USP but excellent execution.

---

## What Makes This Cool

Not a data feature bolted on. The approach is:
- **Craft over cleverness** — a site that loads fast, is always accurate, and
  has clean typography is itself a signal of competence.
- **cv.json** — career history served as well-structured JSON at `/cv.json`. Not
  linked from the homepage. For anyone who goes looking. Genuinely on-brand for a
  data person without performing it.
- **The bio should be human** — "beekeeper, self-taught DJ, freediver" is more
  memorable than "analytical problem-solver." The writing IS the personality.

## Approaches Considered

### Approach A: Pure Static HTML/CSS
Single HTML file. No framework, no build step. Nginx serves directly.
Effort: S | Risk: Low | Completeness: 7/10
Pros: fastest possible, zero deps, trivial Pi deploy
Cons: content updates require HTML edits; starts fresh from timeline.csv

### Approach B: Astro Static Site (CHOSEN)
Static site generator. Content in YAML. Same nginx output.
Effort: S-M | Risk: Low | Completeness: 9/10
Pros: content-data separation, easy to update roles/projects via YAML,
      cv.json generated at build time from same data source, extensible
Cons: one-time build setup (Node needed locally), small build step before deploy

### Approach C: FastAPI + Jinja2 (Python)
Replace Dash with thin Python server. Stays in Python ecosystem.
Effort: S | Risk: Low | Completeness: 7/10
Pros: familiar, can add server-side logic later
Cons: still a running process; more moving parts than static

## Recommended Approach

**Astro static site** — content as YAML via Content Collections, output as static HTML, nginx on Pi.

### Architecture (updated by /plan-eng-review)

```
DATA FLOW
─────────────────────────────────────────────────────
src/content/cv.yaml                  ← edit this to update experience/education
src/content/projects.yaml            ← edit this to update projects
        │
        ▼
src/content/config.ts                ← Zod schema validation (build-time, free)
  defineCollection({ type: 'data',      Any typo in YAML = loud build error
    schema: z.object({...}) })
        │
        ├──► src/pages/index.astro   ← getEntry('cv', 'data') + getCollection('projects')
        │    (Intro, Experience,         groupBy company for Experience section
        │    Projects, Education
        │    components)
        │
        └──► src/pages/cv.json.ts    ─── GET /cv.json
             return new Response(        { name, role, location, contact,
               JSON.stringify({...}),      experience, education, projects,
               { headers: {...} })         generated: "2026-03-25T..." }
─────────────────────────────────────────────────────

BUILD PIPELINE
─────────────────────────────────────────────────────
npm run build
    │
    ├── [prebuild]  ← none needed (Content Collections handles YAML natively)
    ├── [astro build]
    │       ├── validates cv.yaml + projects.yaml against Zod schemas
    │       ├── renders index.html
    │       └── renders cv.json (from src/pages/cv.json.ts)
    └── [postbuild] scripts/validate.mjs
            ├── checks dist/index.html exists + contains "Scott Tomlins"
            └── checks dist/cv.json is valid JSON with expected keys
─────────────────────────────────────────────────────
```

### Pi nginx config (updated)

```nginx
# Redirect www → apex
server {
    listen 80;
    server_name www.stomlins.com;
    return 301 https://stomlins.com$request_uri;
}

# Main site
server {
    listen 80;
    server_name stomlins.com;
    root /home/pi/stomlins.com;
    index index.html;

    # Cache static assets
    location ~* \.(css|js|png|ico|svg|webmanifest)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /cv.json {
        add_header Content-Type application/json;
    }
}
```
